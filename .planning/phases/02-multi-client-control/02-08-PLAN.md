    ```go
    // LeaseAcquiredTotal counts lease acquisition events (CTL-02).
    // No device_serial label per D-18 (per-device labels deferred to Phase 3).
    LeaseAcquiredTotal = prometheus.NewCounter(prometheus.CounterOpts{
        Name: "gateway_lease_acquired_total",
        Help: "Total lease acquisition events.",
    })

    // LeaseReleasedTotal counts lease release events by reason (CTL-02).
    // No device_serial label per D-18.
    LeaseReleasedTotal = prometheus.NewCounterVec(prometheus.CounterOpts{
        Name: "gateway_lease_released_total",
        Help: "Total lease release events by reason (expired, client_released, admin_revoked, device_gone).",
    }, []string{"reason"})

    // WSFramesSentTotal counts WebSocket frames written to clients (OBS-01).
    // Differs from FramesEmittedTotal: that counts Hub fan-out sends (which may
    // be buffered or dropped), this counts actual WS write calls. The two should
    // converge under normal operation but diverge when slow consumers are evicted.
    WSFramesSentTotal = prometheus.NewCounterVec(prometheus.CounterOpts{
        Name: "gateway_ws_frames_sent_total",
        Help: "Total WebSocket frames successfully written to clients, by stream type.",
    }, []string{"stream"})

    // HubViewersActive tracks active viewers per stream type (OBS-01).
    // No device_serial label per D-18 (per-device labels deferred to Phase 3).
    HubViewersActive = prometheus.NewGaugeVec(prometheus.GaugeOpts{
        Name: "gateway_hub_viewers_active",
        Help: "Number of active viewers currently subscribed to a stream, by stream type.",
    }, []string{"stream"})
    ```

  <action>
    **Instrument LeaseManager (internal/session/lease.go):**

    The LeaseManager currently has no `obs` imports. Add the import and instrument the lifecycle events:

    1. **Acquire** — After creating the lease and setting the timer, increment `obs.LeaseAcquiredTotal`:
       ```go
       obs.LeaseAcquiredTotal.Inc()
       ```
       No structural changes needed — LeaseAcquiredTotal is a plain counter with no labels (per D-18, device_serial labels deferred to Phase 3).

    2. **reapLockedLocked** — After signaling the release channel, increment `obs.LeaseReleasedTotal` with the reason:
       ```go
       obs.LeaseReleasedTotal.WithLabelValues(string(reason)).Inc()
       ```
       This covers all release paths: expired, client_released, admin_revoked, device_gone.

    **Instrument Hub (internal/session/hub.go):**

    1. **Subscribe** — After adding the viewer to the map, increment `obs.HubViewersActive`:
       ```go
       obs.HubViewersActive.WithLabelValues(h.stream).Inc()
       ```
       No structural changes needed — HubViewersActive uses only the `stream` label per D-18 (device_serial labels deferred to Phase 3).

    2. **Unsubscribe (the closure returned from Subscribe)** — Decrement the gauge:
       ```go
       obs.HubViewersActive.WithLabelValues(h.stream).Dec()
       ```

    3. **evict** — Decrement the gauge (viewer is being forcefully removed):
       ```go
       obs.HubViewersActive.WithLabelValues(h.stream).Dec()
       ```

    4. **Run (fan-out loop)** — After successfully writing to a viewer's send channel, increment `obs.WSFramesSentTotal`:
       ```go
       case v.send <- msg:
           v.consecutiveDrops = 0
           obs.FramesEmittedTotal.WithLabelValues(h.stream).Inc()
           obs.WSFramesSentTotal.WithLabelValues(h.stream).Inc()
       ```
       Note: `WSFramesSentTotal` differs from `FramesEmittedTotal` in scope — `FramesEmittedTotal` counts Hub fan-out sends (which includes the counter already there), while `WSFramesSentTotal` counts actual WS writes. For now they track the same count at the Hub level. The distinction becomes meaningful if we add a WS-level write counter in the API layer later (Plan 07's CloseRead fix enables this).

    **Important:** Add `"github.com/pelni/adb-gateway/internal/obs"` import to lease.go (hub.go already imports obs). No structural changes to LeaseManager or HubOpts are needed — all four new collectors use no or low-cardinality labels only (per D-18, device_serial labels are Phase 3).
