### Key Changes

- **`keyLimiter`** (renamed from `apkKeyLimiter`): generic per-key token bucket, reused for APK install rate limit and new file-app write rate limit.
- **`requireWriteRateLimit`** middleware: wraps chi routes with per-owner-key rate limiting; rejects with 429 `RATE_LIMITED`.
- **`FilesDispatcher`**: 4 HTTP method dispatchers (Get/Post/Patch/Delete) routing by `?op=` param. Unknown ops return 400 `UNSUPPORTED_OP`. DELETE rejects `?op=` values (use `?recursive=1` instead). Phase 3 compat: no `?op=` routes to original DownloadFile/UploadFile/DeleteFile.
- **Router wiring**: `/files` uses dispatcher for all methods; `/apps` subtree under `/{serial}` with rate-limited write ops (backup, uninstall); read ops (list, details, apk export) are not rate-limited.
- **`config.yaml.example`**: Documents all Phase 3 + 03.1 config knobs with defaults and comments. No new koanf keys added (per CONTEXT.md scope).
