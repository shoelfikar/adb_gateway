## What Was Built

### Error Sentinels (D-ERR-01)

Seven new `DomainError` sentinels added to `internal/api/errors.go`:

| Sentinel | HTTP Status | Code |
|----------|-------------|------|
| `ErrRenameCrossFS` | 409 Conflict | `RENAME_CROSS_FS` |
| `ErrRenameFailed` | 500 Internal Server Error | `RENAME_FAILED` |
| `ErrInvalidPackage` | 400 Bad Request | `INVALID_PACKAGE` |
| `ErrPackageNotFound` | 404 Not Found | `PACKAGE_NOT_FOUND` |
| `ErrUninstallFailed` | 500 Internal Server Error | `UNINSTALL_FAILED` |
| `ErrBackupFailed` | 500 Internal Server Error | `BACKUP_FAILED` |
| `ErrListFailed` | 500 Internal Server Error | `LIST_FAILED` |

`UNSUPPORTED_ENTRY` intentionally excluded from errors.go per D-FB-09 (NDJSON-only, never an HTTP status).

### FileShellRunner Interface Extension

Added `ShellV2Stream(ctx, serial, cmd) (stdout, stderr io.ReadCloser, exit <-chan int, err error)` to `FileShellRunner` interface. This enables Wave 1 backup/uninstall/list/rename handlers to use `ShellV2Stream` through the same test injection seam as existing file handlers. `*adb.HostServices` already satisfies the extended interface.

### fakeFileRunner ShellV2Stream Stub

Added no-op `ShellV2Stream` stub with `shellV2Calls []string` field for call recording. Returns empty closed readers + buffered exit(0) channel. Existing Phase 3 tests pass unchanged.

### DeviceEntry.WriteInFlight

Added `WriteInFlight atomic.Bool` to `DeviceEntry` for Phase 03.1 write-op single-flight gate. Independent of `InstallInFlight` so APK install does not block backup/uninstall. CAS-only access per Pitfall 9.

### Package Name Validator (REQ-AM-PKG-VALIDATE)

`validatePackage(w, r)` reads chi URL param "pkg" and validates against anchored regex `^[a-zA-Z][a-zA-Z0-9_]*(\.[a-zA-Z][a-zA-Z0-9_]*)+$` before any shell call. 256-byte belt-and-braces cap. Invalid inputs receive 400 `INVALID_PACKAGE`.

### LS Parser (D-FB-03)

`ParseLSLine(line, dir)` and `ParseLSOutput(out, dir)` parse toybox `ls -lA --time-style=full-iso` output into `Entry` structs with exact D-FB-03 shape. Handles regular files, directories, symlinks, filenames with spaces, short-iso layout fallback. `permToOctal` converts permission strings like `-rw-rw----` to octal `"0660"`.
