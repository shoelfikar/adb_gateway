## What Was Built

### ExportAPK Handler (D-AM-06/07)

`ExportAPK` / `ExportAPKForTest` -- GET /devices/{serial}/apps/{pkg}/apk

- `validatePackage(w, r)` called BEFORE any shell call (REQ-AM-PKG-VALIDATE invariant)
- Runs `pm path 'pkg'` (shellQuote single-quoted) to discover APK paths on device
- Empty `pm path` output (0 paths) returns 404 PACKAGE_NOT_FOUND
- I1: `dumpsys package 'pkg'` via ParseDumpsysPackage to extract version code for filename; best-effort -- failure leaves `<pkg>-0.apk`
- Single APK (1 path): Content-Type `application/vnd.android.package-archive`, filename `<pkg>-<versionCode>.apk`, streamed directly via SyncPullWriter
- Split APK (2+ paths): Content-Type `application/x-tar`, filename `<pkg>-<versionCode>.tar`, tar containing ALL parts (D-AM-07: NO base-only fallback)
- Read-op: does NOT acquire WriteInFlight (Pitfall 9)
- Split-APK buffers each part in memory before writing tar header (Size must be known upfront); typical APK parts <100 MiB
